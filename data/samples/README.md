# Samples

These four images are a small manual stress-test set.

They are kept in the repo mainly to sanity-check edge cases that are not well covered by the Horizon-UAV dataset, especially rotated and near-vertical horizons.

They are **not** part of the main automated evaluation run in [tools/evaluate.py](/Users/jurajuhlar/Documents/Code/personal/etdh-horizon-detection/tools/evaluate.py), which currently evaluates against `data/horizon_uav_dataset/`.

Use them for:

- quick visual checks while iterating on an attempt
- demo outputs saved under each attempt's `outputs/` or `results/`
- manual validation of rotation-invariant behavior
